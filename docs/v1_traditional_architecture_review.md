# 架构演进复盘记录：V1 (传统 CV 切割 + CNN 分类)

## 1. 时代背景与核心目的

在深度学习全面接管 OCR 领域之前，OCR 这门技术几乎等同于“图像文字切分艺术”。V1 作为整个教学沙箱的第一代基石，其核心目的是通过代码全景展示出**传统切分管线（Pipeline）**的工作原理，同时刻意暴露它的致命痛点，以此反衬后续端到端序列处理模型（V2 CRNN）出现的历史必然性。

## 2. 架构骨架：流水线式的断代解耦

传统 OCR 严格遵循“先切再认，互不干涉”的解耦哲学：

### 第一步：CV 物理切分 (`utils/image_processing.py`)

模型本身并不具备寻找文字的能力，所有的一切都依赖于特征工程硬编排。由于算力低下，系统必须努力将多文字降维为独立字符图像：

1. **灰度与二值化 (Otsu's Thresholding)**：抛离色彩信息，寻找最佳阈值形成黑白图像，分离前景与背景。
2. **形态学修补 (Morphology - Close)**：通过膨胀再腐蚀，将断裂的笔画强行桥接回一个整体的物理联通域。
3. **垂直投影切割 (Vertical Projection)**：将二值图在竖直方向上累加像素，通过寻找“像素谷底（波流低洼处）”作为字符间的留白，使用 OpenCV 直接裁剪出单字切片（BBox）。

### 第二步：孤岛式的模型推断 (`models/cnn_classifier.py`)

这是一个最简单的 2 枚 Conv2D + MaxPool 组成的微型 CNN 模型。
它只认识被切好的、长宽固定的小图。将其送入全连接层后通过 `CrossEntropy` 分类输出 0-9 之一的任务。它对左边出现了什么、右边出现了什么**一无所知**。

## 3. 终局与痛点分析：木桶效应

在我们的 V1 UI 界面中，当输入一张轻微粘连的测试图片时：

- 由于字母粘连，**垂直投影**在谷底无法回落至 0，导致算法切不开或者切出包含两三个重影的乱码块。
- 一旦物理切分发生大面积惨案，不管后段的 CNN 拥有多么逆天的 99% 的单子准确率，全图正确率立刻跌至 0。

**结论：** 传统切分的硬切割极度害怕**粘连、断笔、极度倾斜与噪声**。切割的上限即是 OCR 的上限。在工业界面对复杂真实环境彻底宣判死刑，呼唤着不需要切分即可序列识别的霸主——**CRNN + CTC 联合体 (V2 的降临)**！
