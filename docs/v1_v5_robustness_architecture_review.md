# 架构演进复盘记录：V1-V5 鲁棒性与防御体系升级 (Robustness Enhancements)

**日期:** 2026-03-13
**背景:** 本次演进旨在解决原始“实验室沙箱”OCR 模型在面临真实世界复杂输入（倾斜、阴影、光照不均、严重粘连、多行堆叠、透视畸变）时的脆弱性。通过引入一系列前置图像处理与几何形变学习机制，大幅提升了全系模型的泛化与抗干扰能力。

---

## 一、 影响范围全局化的基础防线

### 1. 霍夫倾斜校正 (Global Skew Correction)
*   **目标:** 解决因用户拍摄角度导致的文本行整体倾斜，防止垂直投影重叠和序列模型特征（ Attention / CNN 感受野）撕裂错位。
*   **实现机制:**
    *   被整合入全局共享的 `utils/image_processing.py : unified_enhance_image` 前置拦截器中。
    *   **核心算子:** `cv2.Canny` 边缘检测 + `cv2.HoughLinesP` 概率霍夫变换提取线段。
    *   **兜底机制:** 当霍夫变换失效（无法提取足够长的水平线段）时，启用 `cv2.minAreaRect` 获取所有文本像素的最小外接矩形角度作为降级兜底方案。
    *   **校正动作:** 对偏移角度 $> 0.5^\circ$ 的图像，使用 `cv2.warpAffine` 进行中心旋转展平，并强制使用纯白色 (`borderValue=255`) 填充旋转产生的越界黑边，避免产生干扰 CNN 的虚假高频边缘。
*   **收益:** 全局 V1-V5 模型均享有“端正输入”的红利。

## 二、 V1 传统工程学组件专项重构

为了强化教学沙箱对数字图像处理原教旨主义的展示，针对 V1 极度依赖垂直投影的弱点进行了定向防御：

### 1. 自适应阈值 (Adaptive Thresholding)
*   **痛点:** 传统全局 Otsu 阈值在一刀切判定“阴阳脸”、“局部反光”或“渐变阴影”图像时，会导致大片字符断裂甚至直接消失在背景中。
*   **实现机制:**
    *   引入 `cv2.adaptiveThreshold`，通过计算高斯邻域内的局部加权平均值来动态决定每一个像素的局部最优二值化阈值。
    *   在 V1 的侧边栏 UI 中，将核心影响因子 `Block Size`（奇数窗口大小）和 `C`（高斯均值补偿系数）彻底暴露，作为“防御阵地调参”的教学看板。

### 2. 启发式滴水盲切 (Heuristic Water Drop Splitting)
*   **痛点:** 多次形态学膨胀/腐蚀依然无法分离的严重粘连字符，会导致垂直投影产生一个跨度极长、没有明显谷底的单一宽峰，直接宣判单个连通域识别毁灭。
*   **实现机制:**
    *   放弃计算量巨大的物理重力模拟滴水算法，改用具备 O(n) 时间复杂度的“启发式寻峰寻底”。
    *   对 `宽度 > 高度 * 1.2` 的可疑“巨无霸”连通域触发该防线。
    *   在该连通域的内部核心段（避开两端首尾 20% 防误切），沿 X 轴扫描并计算每一列顶部和底部的最近黑色素点的深度厚度。
    *   找出上下文物理间距最薄弱的点（即上下轮廓收缩最为接近处），进行竖直方向的一刀切截断。并在 UI 上用红色折线可视化还原斩断路径。

## 三、 V2-V5 序列模型工业化切割升级

纯粹的序列模型（CRNN/Seq2Seq/Transformer）虽然对单维度字符连接具备免疫能力，但极其畏惧背景大片噪点的压缩效应和多行文本的 Y 轴重叠。

### 1. 结构化 BBox 提取 (Maximum Connected Component Extraction)
*   **痛点:** 原先简单粗暴的 `cv2.findNonZero` + `cv2.boundingRect` 会将画面角落里仅仅 1px 的反光噪点囊括进 BBox，导致原本的有效文字区域在被强行 Resize 到 `32x256` 时被挤压成极细的面条。
*   **实现机制:**
    *   新增 `extract_text_lines` 管道，改用 `cv2.findContours` 提取出画面中所有的物理连通轮廓。
    *   实施面积过滤硬规则（目前硬编码为滤除面积占比 $< 0.1\%$ 的飞点或固定微小绝对面积），只保留真正代表油墨笔划的主体轮廓参与外接矩形的计算。

### 2. 多行文本感知与剥离 (Line Segmentation via Horizontal Projection)
*   **痛点:** V2-V5 模型设定为单行长条形输入处理单元，面对多段落或分行文本直接输入时会造成毁灭性重叠解析。
*   **实现机制:**
    *   基于上一步清洗后留下的有效轮廓掩码，计算水平方向累加的投影柱状图（Horizontal Projection）。
    *   通过寻找水平投影上的零值/低值波谷（并设立行间噪点容忍阈值），精准切分出多行物理行条 (Strips)。
    *   在前端返回 List 切片结果后，于 `pages/v4_transformer_joint.py` 中引入单选按钮（Radio），允许用户直观地选中多行文本中的任意一行，进而下探到 Attention Matrix 和 CTC 热力图对该行进行局部切片分析。

## 四、 V4 混合架构新增前置形变网络 (STN)

### 1. 空间变换网络 (Spatial Transformer Network)
*   **痛点:** 基于 CNN-特征平展-Transformer 的直筒结构在面临复杂透视畸变、扭曲数字时，特征提取能力受制于固定卷积核的几何局限性。
*   **实现机制:**
    *   在 `models/v4_transformer_joint.py` 内硬编码实现了基于 `Localization Network` + `Affine Matrix Regressor` + `grid_sample` 的原生 PyTorch 微型 STN。
    *   为保障教育沙箱对旧有训练用户的平滑过渡体验，该模块被标记为条件接入 (`use_stn=True/False`)。
    *   在 UI 层暴露该模块的开启开关，开启后其将伴随主网络端到端训练出独立的参数文件 (`best_v4_stn_joint.pth`)，使得网络能够产生自我纠正透视甚至微小扭曲内禀的能力。
